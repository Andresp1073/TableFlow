# Backend Architecture

**Last updated:** 2026-07-04

## Overview

The backend is a **Modular Monolith with Clean Architecture boundaries**. Each feature module is self-contained with its own controller, service, repository, validators, and types. Cross-cutting concerns (auth, RBAC, logging, error handling) are handled by middleware in the `common/` layer.

## Directory Responsibilities

### `prisma/`

Database schema and migrations.

| File/Dir | Responsibility |
|----------|----------------|
| `schema.prisma` | All models, enums, relations, and indexes. Single source of truth for the data model. |
| `migrations/` | Migrations generated by Prisma. One migration per logical change. |
| `seed.ts` | Idempotent seed script for development and testing. Creates default roles, admin user, and sample data. |

---

### `src/config/`

Application configuration loaded from environment variables.

| File | Responsibility |
|------|----------------|
| `env.ts` | Zod schema for environment variables. Validated at startup. |
| `database.ts` | Prisma client instantiation. |
| `auth.ts` | JWT secret, token expiration, bcrypt cost factor. |
| `logger.ts` | Logger configuration (Pino/Winston). |
| `cors.ts` | CORS origin whitelist. |
| `swagger.ts` | Swagger/OpenAPI configuration. |

```typescript
// env.ts pattern — validate all env vars at startup
export const env = z
  .object({
    NODE_ENV: z.enum(['development', 'testing', 'production']),
    PORT: z.coerce.number().default(3000),
    DATABASE_URL: z.string().url(),
    JWT_SECRET: z.string().min(32),
    JWT_EXPIRES_IN: z.string().default('15m'),
    REFRESH_TOKEN_EXPIRES_IN: z.string().default('7d'),
    BCRYPT_SALT_ROUNDS: z.coerce.number().default(12),
    CORS_ORIGIN: z.string().url(),
    SMTP_HOST: z.string(),
    SMTP_PORT: z.coerce.number(),
    SMTP_USER: z.string(),
    SMTP_PASS: z.string(),
  })
  .parse(process.env);
```

### `src/lib/`

Third-party library initialization.

| File | Responsibility |
|------|----------------|
| `prisma.ts` | Prisma client singleton with connection pooling. |

// prisma.ts pattern
const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

### `src/modules/`

All feature modules. Each module follows a strict internal structure (see [module-architecture.md](./module-architecture.md)).

Each module contains:

| File | Responsibility |
|------|----------------|
| `{feature}.controller.ts` | HTTP handler: extracts params, delegates to service, returns response. |
| `{feature}.service.ts` | Business logic. Pure functions. No HTTP or DB awareness. |
| `{feature}.repository.ts` | Data access. Prisma queries. Returns domain entities. |
| `{feature}.routes.ts` | Route definitions for the module. |
| `{feature}.validator.ts` | Zod schemas for request validation. |
| `{feature}.dto.ts` | Data Transfer Objects (input/output contracts). |
| `{feature}.types.ts` | Module-specific types and enums. |
| `{feature}.interfaces.ts` | Interfaces (repository contract, service contract). |
| `index.ts` | Module public API. Exports routes, service interface. |
| `__tests__/` | Unit and integration tests. |

---

### `src/common/`

Cross-cutting concerns shared across all modules.

| File/Dir | Responsibility |
|----------|----------------|
| `middleware/auth.middleware.ts` | JWT verification, user extraction. |
| `middleware/rbac.middleware.ts` | Permission check against required permission. |
| `middleware/validate.middleware.ts` | Zod schema validation for body/params/query. |
| `middleware/rateLimiter.middleware.ts` | Rate limiting by IP and user. |
| `middleware/errorHandler.middleware.ts` | Global error handler. Catches all errors, formats response. |
| `middleware/audit.middleware.ts` | Logs sensitive operations to audit trail. |
| `middleware/requestLogger.middleware.ts` | Logs every request: method, path, status, duration. |
| `errors/AppError.ts` | Base error class and derived error classes. |
| `interfaces/` | Shared interfaces (pagination, response format). |
| `types/` | Shared types (enums, utility types). |
| `utils/` | Shared utilities (date formatting, token generation, validation helpers). |

### `src/routes/`

Route aggregator that mounts all module routes.

```typescript
// routes/index.ts pattern
const router = Router();

router.use('/auth', authRoutes);
router.use('/reservations', authenticate, reservationRoutes);
router.use('/tables', authenticate, tableRoutes);
router.use('/customers', authenticate, customerRoutes);
router.use('/restaurants', authenticate, restaurantRoutes);
router.use('/users', authenticate, rbac('users.list'), userRoutes);
router.use('/roles', authenticate, rbac('roles.list'), roleRoutes);
router.use('/notifications', authenticate, notificationRoutes);
router.use('/reports', authenticate, rbac('reports.view'), reportRoutes);
router.use('/dashboard', authenticate, dashboardRoutes);
router.use('/settings', authenticate, settingsRoutes);
router.use('/audit', authenticate, rbac('audit.read'), auditRoutes);

export default router;
```

### `src/app.ts`

Express application setup.

```typescript
// app.ts pattern
const app = express();

app.use(helmet());
app.use(cors(corsConfig));
app.use(express.json({ limit: '1mb' }));
app.use(requestLogger);

app.use('/api/v1', routes);

app.use(errorHandler);

export default app;
```

## Module Communication Rules

See [dependency-rules.md](./dependency-rules.md) for the complete dependency rules.

**Summary:**
- Modules communicate through **service interfaces**, never through repositories.
- Module A can call Module B's service only through the interface defined in Module B's `index.ts`.
- The `common/` layer is accessible to all modules and contains no module-specific logic.
- Circular dependencies between modules are **strictly prohibited**.

## Middleware Pipeline

```
Request
  │
  ▼
helmet()                                    → Security headers
cors()                                      → CORS check
express.json()                              → Body parsing
requestLogger                               → Log request
authenticate (JWT verify)                   → Auth check
  │
  ├─ Public routes: skip auth
  └─ Protected routes: continue
      │
      ▼
rbac('permission.name')                     → Permission check
  │
  ├─ Denied → 403
  └─ Allowed → continue
      │
      ▼
validate(schema)                            → Zod validation
  │
  ├─ Invalid → 400
  └─ Valid → continue
      │
      ▼
Controller → Service → Repository → Prisma → MySQL
      │
      ▼
Response
```
