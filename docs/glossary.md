# Glossary

| Term | Definition |
|------|------------|
| **Access Token** | Short-lived JWT token used to authenticate API requests. |
| **Audit Log** | Immutable record of system events used for security and compliance. |
| **Automatic Table Assignment** | System algorithm that assigns the most suitable available table based on party size and preferences. |
| **Average Turn Time** | The average time a table is occupied from check-in to check-out. |
| **Branch** | A single physical restaurant location under a parent organization. |
| **Cancellation Window** | The minimum advance time required before a reservation can be canceled without penalty. |
| **Check-In** | The process of recording a guest's arrival at the restaurant. |
| **Check-Out** | The process of recording a guest's departure and releasing the table. |
| **Cleaning Buffer** | The configured time interval between reservations on the same table for cleaning and resetting. |
| **Confirmation Code** | A unique alphanumeric identifier assigned to every reservation. |
| **Cover** | A single seated guest; used in capacity and occupancy calculations. |
| **Customer Profile** | A database record containing customer contact information, preferences, and visit history. |
| **Dashboard** | The main landing page showing real-time operational metrics. |
| **Floor Plan** | A visual representation of the restaurant's table layout, organized by zones. |
| **Grace Period** | The configured time after a reservation time during which a late arrival is stillaccepted before being marked as no-show. |
| **JWT** | JSON Web Token — a compact, URL-safe token format used for authentication. |
| **No-Show** | A reservation where the guest does not arrive and is not canceled within the grace period. |
| **Occupancy Rate** | The percentage of total seating capacity that is occupied over a given period. |
| **Party Size** | The number of guests in a single reservation. |
| **Peak Hours** | Time periods with the highest reservation volume, identified through analytics. |
| **POS** | Point of Sale — a system used for processing transactions in restaurants. |
| **RBAC** | Role-Based Access Control — a permission model where access is granted based on user roles. |
| **Refresh Token** | A longer-lived token used to obtain new access tokens without re-authentication. |
| **Refresh Token Rotation** | The practice of issuing a new refresh token each time one is used, invalidating the previous one. |
| **Reservation Slot** | A discrete time interval during which a reservation can be scheduled. |
| **Restaurant Administrator** | A user role with full management access to one or more restaurant branches. |
| **Slot Interval** | The time increment used for reservation availability (e.g., 15, 30, or 60 minutes). |
| **Soft Delete** | A deletion method where the record remains in the database but is marked as inactive. |
| **Table Merge** | Combining two or more adjacent tables into one larger table for accommodating larger parties. |
| **Table Zone** | A designated area within a restaurant (e.g., patio, bar, indoor, VIP room). |
| **Turn Time** | The total time a table is occupied by a single party from seating to departure. |
| **Walk-In** | A guest who arrives without a prior reservation and is seated if a table is available. |
| **WCAG** | Web Content Accessibility Guidelines — standards for making web content accessible to people with disabilities. |
| **Zone** | A section of the restaurant floor plan grouping tables by location or type. |
